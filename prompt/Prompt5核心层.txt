请审查 core/ 目录下的所有文件，做以下优化（不新增文件）：

1. core/models.py：
   - 检查所有涉及价格、数量、金额的字段，是否使用了 Decimal 类型。如果用的是 float，请替换为 Decimal，并添加精度控制。
   - 所有 Model 类应有完整的 __repr__ 和基本数据校验（价格不能为负、数量不能为 0 等），如果没有请添加。

2. core/event_bus.py：
   - 一个订阅者的回调抛异常不应影响其他订阅者。检查是否有 try-except 包裹每个回调调用。没有请添加，异常时记录日志但不中断事件传播。
   - 关于是否需要支持异步事件：如果当前使用场景都是同步的，保持同步即可，不要为了"以后可能用到"而过度设计。

3. core/engine.py：
   - 是否有优雅退出机制？捕获 SIGINT/SIGTERM 后应按顺序：暂停策略 → 撤未完成订单 → 断开连接 → 保存状态。如果没有请添加。
   - 引擎启动时是否有自检流程（检查配置完整性、交易所连接可用性）？任一失败应拒绝启动并给出明确错误。没有请添加。

4. core/exceptions.py：
   - 异常类是否覆盖关键场景：配置错误、网络错误、交易所 API 错误、风控拦截、数据异常。缺失的补上，但不要过度细分。

5. core/logger.py：
   - 是否有日志轮转配置（按天或按大小）？没有请加上。
   - 是否支持结构化日志（JSON 格式）？如果当前只是文本日志，添加一个可选的 JSON formatter，方便后续分析。但不强制所有地方都用 JSON，保持灵活。

6. core/enums.py：
   - 审查枚举定义是否完整覆盖了：订单方向、订单类型、订单状态、持仓方向、K 线周期等。缺失的补上。
